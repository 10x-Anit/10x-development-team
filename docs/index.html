Dashboard for Your Existing Data

Developer · Has data in Supabase, needs a UI fast

Marcus has 50,000 rows of product analytics in Supabase. He needs a dashboard to visualize it — but doesn't want to spend a week building one from scratch.

1
Start a project
"Build me a dashboard that shows product analytics — page views, conversions, revenue by product." Picks MVP scope.
2
Connect the data
Runs :connect-data Supabase. The plugin sets up the Supabase client, creates API proxy routes, wires hooks into the pages.
3
Tweak the UI live
"Change the cards to a data table with sorting." :modify-ui change cards to sortable table. Done in seconds.
4
Add real-time updates
"Make the revenue chart update live." The plugin adds Supabase Realtime subscriptions — data flows in without page refresh.
Result

A live analytics dashboard connected to existing Supabase data. Stat cards, sortable tables, real-time charts. Built in under an hour, not a week.

Scale an Existing App with New Features

Startup CTO · App is live, needs to ship faster

Aisha's SaaS has 500 users. She needs to add Stripe payments, email notifications, and a settings page. Her team is small. She uses the plugin to move 3x faster.

1
Resume the existing project
:resume — the plugin loads the project from persistent memory. It knows every file, every feature, every decision from previous sessions.
2
"Add Stripe subscription payments"
:add-feature users can subscribe to monthly plans with Stripe. Backend Dev creates Stripe webhooks + checkout. Frontend Dev adds the pricing page.
3
"Add email notifications"
:add-feature send welcome email on signup and invoice email on payment. Backend Dev sets up Resend with React Email templates.
4
Add the settings page
:add-page account settings with profile, billing, and notification preferences. Uses existing auth context and component registry.
5
Review and deploy
:review runs QA checks. :deploy updates the CI/CD pipeline. Feature map tracks exactly which files changed for each feature.
Result

3 major features added to a live app in one session. The plugin knew the existing codebase, reused components, and updated the feature map. No breaking changes. Shipped same day.
